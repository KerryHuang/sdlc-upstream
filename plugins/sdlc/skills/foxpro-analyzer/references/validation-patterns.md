# FoxPro 驗證模式詳細對照

## 必填欄位驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `EMPTY(欄位)` | 檢查欄位是否為空 |
| `LEN(TRIM(欄位)) = 0` | 檢查去除空白後長度為 0 |
| `ISNULL(欄位)` | 檢查是否為 NULL |
| `欄位 = ""` | 直接比對空字串 |
| `欄位 = SPACE(N)` | 比對 N 個空白字元 |

### 轉換規則

```markdown
| 規則編號 | 欄位 | 規則描述 | 錯誤訊息 |
|---------|------|---------|---------|
| VR-XXX | {欄位名稱} | 必填 | 「請輸入{欄位中文名稱}」 |
```

## 特殊字元驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `ASC(SUBSTR(欄位, N, 1))` | 取得第 N 個字元的 ASCII 碼 |
| `ISALPHA(欄位)` | 檢查是否為字母 |
| `ISDIGIT(欄位)` | 檢查是否為數字 |
| `AT("特殊字元", 欄位)` | 搜尋特殊字元位置 |

### 常見禁用字元

| ASCII 範圍 | 對應字元 |
|-----------|---------|
| 32 | 空格 |
| 33-41 | ! " # $ % & ' ( ) |
| 46-47 | . / |
| 58-64 | : ; < = > ? @ |
| 91-96 | [ \ ] ^ _ ` |

### 轉換規則

```markdown
| 項目 | 說明 |
|------|------|
| 驗證時機 | 新增/修改時 |
| 驗證邏輯 | 檢查欄位內容是否包含禁用特殊字元 |
| 禁用字元 | 空格、!、"、#、$、%、&、'、(、)、.、/ |
| 錯誤訊息 | 「請勿使用特殊字元」 |
```

## 唯一性驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `SEEK(值, 別名)` | 在索引中搜尋 |
| `LOCATE FOR 條件` | 循序搜尋 |
| `SELECT ... WHERE` | SQL 查詢 |
| `INDEXSEEK(值)` | 索引搜尋 |

### FoxPro SEEK 語法解析

```
SEEK(搜尋值, "資料表別名")
IF FOUND()
   * 找到重複資料
   MESSAGEBOX("資料已存在")
ENDIF
```

### 轉換規則

```markdown
| 項目 | 說明 |
|------|------|
| 驗證時機 | 新增時 |
| 驗證邏輯 | 檢查資料表中是否已存在相同值（排除已刪除資料） |
| 查詢條件 | {欄位} = {輸入值} 且 刪除標記 ≠ 'Y' |
| 錯誤訊息 | 「{欄位描述}已存在」 |
```

## 關聯資料限制

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `SELECT 子表` | 切換到子表 |
| `LOCATE FOR 外鍵 = 值` | 搜尋關聯資料 |
| `COUNT FOR 條件` | 計算關聯筆數 |
| `RECCOUNT()` | 取得記錄數 |

### FoxPro 刪除限制語法

```
SELECT 子表
LOCATE FOR 外鍵 = 主表.主鍵
IF FOUND()
   MESSAGEBOX("子表資料已存在，不能刪除")
   RETURN .F.
ENDIF
```

### 轉換規則

```markdown
| 主表 | 關聯表 | 限制說明 | 錯誤訊息 |
|------|-------|---------|---------|
| {主表名稱} | {關聯表名稱} | 有關聯資料時不可刪除 | 「{關聯資料描述}已存在，不能刪除」 |
```

## 數值範圍驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `BETWEEN(值, 下限, 上限)` | 範圍檢查 |
| `值 >= 下限 AND 值 <= 上限` | 條件範圍 |
| `INLIST(值, 選項1, 選項2)` | 列舉檢查 |

### 轉換規則

```markdown
| 規則編號 | 欄位 | 規則描述 | 錯誤訊息 |
|---------|------|---------|---------|
| VR-XXX | {欄位} | 必須介於 {下限} 至 {上限} | 「{欄位}必須介於 {下限} 至 {上限}」 |
```

## 加總驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `SUM(欄位)` | 直接加總 |
| `CALCULATE SUM(欄位)` | 計算加總 |
| `SELECT SUM(欄位) FROM 表格` | SQL 加總 |

### FoxPro 加總驗證語法

```
SELECT 子表
SUM 比例欄位 TO 總和變數 FOR 外鍵 = 值
IF 總和變數 <> 100
   MESSAGEBOX("比例合計必須等於 100%")
ENDIF
```

### 轉換規則

```markdown
| 項目 | 說明 |
|------|------|
| 驗證時機 | 新增/修改/刪除後 |
| 驗證邏輯 | 計算該群組所有記錄的{欄位}加總 |
| 預期結果 | 加總應等於 {預期值} |
| 錯誤訊息 | 「{欄位}合計錯誤，請修正」 |

**驗證情境**：

| 加總結果 | 處理方式 |
|---------|---------|
| = 預期值 | 通過驗證 |
| < 預期值 | 顯示警告「合計小於 {預期值}」 |
| > 預期值 | 顯示警告「合計大於 {預期值}」 |
```

## 日期驗證

### FoxPro 常見寫法

| 模式 | 說明 |
|------|------|
| `DATE()` | 取得今天日期 |
| `DATETIME()` | 取得現在時間 |
| `CTOD("日期字串")` | 字串轉日期 |
| `DTOS(日期)` | 日期轉字串 |
| `日期1 > 日期2` | 日期比較 |

### 轉換規則

```markdown
| 規則編號 | 欄位 | 規則描述 | 錯誤訊息 |
|---------|------|---------|---------|
| VR-XXX | {日期欄位} | 不可早於 {基準日期} | 「{欄位}不可早於{基準}」 |
| VR-XXX | {日期欄位} | 不可晚於今天 | 「{欄位}不可晚於今天」 |
```

## 條件必填驗證

### FoxPro 常見寫法

```
IF 條件欄位 = "某值"
   IF EMPTY(目標欄位)
      MESSAGEBOX("當條件為某值時，目標欄位必填")
   ENDIF
ENDIF
```

### 轉換規則

```markdown
| 項目 | 說明 |
|------|------|
| 驗證時機 | 新增/修改時 |
| 前置條件 | 當 {條件欄位} = {特定值} 時 |
| 驗證邏輯 | {目標欄位} 必須有值 |
| 錯誤訊息 | 「當{條件描述}時，請輸入{目標欄位}」 |
```
