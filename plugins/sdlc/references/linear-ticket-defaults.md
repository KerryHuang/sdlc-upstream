# Linear 開票預設值與描述範本

> 供 Linear 開票功能使用，定義各票的 Assignee、狀態、描述格式

## Assignee 與狀態預設值

| 票種 | Assignee | 初始狀態 |
|------|----------|----------|
| PM 票 | 開票人（當前使用者，使用 `"me"`） | Backlog |
| QA 票 | `{QA_assignee}` | Revisable |
| Dev 後端票 | `{backend_assignee}` | Revisable |
| Dev 前端票 | `{frontend_assignee}` | Revisable |

### 重要：狀態指定方式

Linear MCP 用名稱匹配狀態時，同 type 的多個狀態可能導致匹配錯誤（如 `Backlog` 被匹配到同為 backlog type 的其他狀態）。

**必須使用狀態 ID 指定**，避免名稱匹配歧義：

> **注意**：以下狀態 ID 需依實際 Linear 環境替換。

| 團隊 | Backlog ID | Revisable ID |
|------|-----------|-------------|
| `{PM_team}` | `{PM_team_backlog_id}` | `{PM_team_revisable_id}` |
| `{QA_team}` | `{QA_team_backlog_id}` | `{QA_team_revisable_id}` |
| `{Dev_team}` | `{Dev_team_backlog_id}` | `{Dev_team_revisable_id}` |

## 文件 Repo 連結設定

規格書若存放在獨立 docs repo，建票時自動掛為資源連結（`save_issue` 的 `links` 參數）。

| 設定項 | 說明 | 佔位符 |
|--------|------|--------|
| Docs Repo URL | 文件 repo 的 blob URL 前綴 | `{docs_repo_url}` |

> 範例：`https://github.com/KerryHuang/moldplan.docs/blob/main`
> 完整連結 = `{docs_repo_url}/{URL encoded path}`

若 `{docs_repo_url}` 未設定（仍為佔位符），則跳過資源連結，僅在票描述中附文件路徑。

## 操作流程

### 查詢流程
1. 進入頁面 → 載入預設查詢條件
2. 輸入/選擇篩選條件 → 點擊查詢
3. 顯示分頁結果 → 可排序、換頁

### 新增流程
1. 點擊「新增」按鈕 → 開啟空白表單
2. 填寫必填欄位 → 前端即時驗證
3. 點擊「儲存」→ 呼叫 POST API → 成功提示 → 返回列表

### 編輯流程
1. 點擊列表項目 → 載入單筆資料（GET API）
2. 修改欄位 → 前端即時驗證
3. 點擊「儲存」→ 呼叫 PUT API → 成功提示 → 返回列表

### 刪除流程
1. 選取項目 → 點擊「刪除」→ 確認對話框
2. 確認 → 呼叫 DELETE API → 成功提示 → 列表刷新
