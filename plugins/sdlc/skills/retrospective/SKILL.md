# Session 回顧與 Plugin 自我進化

從當前 session 的實際使用體驗中，提取 plugin 本身的改進機會，讓 plugin 持續進化。

## 使用者輸入

```text
$ARGUMENTS
```

- `retrospective` — 回顧整個 session，產出改進報告
- `retrospective --apply` — 產出報告並直接修改 plugin 檔案
- `retrospective {skill-name}` — 只回顧特定 skill 的使用體驗

## 執行流程

### 1. 收集 Session 證據

回顧當前對話中所有與 plugin skills 相關的互動，提取：

| 收集項 | 來源 | 找什麼 |
|--------|------|--------|
| 調用了哪些 skills | 對話中的 `/sdlc:*` 調用 | 使用頻率、順序 |
| 使用者的修正 | 使用者說「不對」「改成」「不是這樣」 | skill 指引不夠明確的地方 |
| 補充說明 | 使用者額外解釋 skill 沒涵蓋的內容 | skill 缺漏的知識 |
| 流程卡點 | 使用者等待、重試、繞路 | 流程不順暢的環節 |
| 手動調整 | 使用者手動修改 skill 產出的內容 | 範本或格式需改進 |
| 好的模式 | 使用者確認「對」「好」「就是這樣」 | 值得固化的做法 |
| 意外發現 | session 中學到的新知識 | 應納入 skill 或 reference |

### 2. 分類改進點

將收集到的證據分為五類：

#### 2.1 Skill 內容改進

skill 的指引不夠清楚、步驟有遺漏、或產出品質不符預期。

| 改進項 | 範例 |
|--------|------|
| 步驟遺漏 | 「規格書寫完忘了檢查 X」 |
| 指引模糊 | 「不知道何時該用 A 還是 B」 |
| 產出格式 | 「表格欄位順序不直覺」 |
| 判斷邏輯 | 「模式偵測條件不夠精確」 |

#### 2.2 範本改進

範本的結構、欄位、範例需要更新。

| 改進項 | 範例 |
|--------|------|
| 章節缺失 | 「BFS 範本應多一個 X 章節」 |
| 欄位調整 | 「這個表格應多一欄 Y」 |
| 範例更新 | 「JSON 範例不符實際結構」 |
| 註解補充 | 「撰寫指南應提醒 Z」 |

#### 2.3 Reference 改進

參考文件的知識需要補充或修正。

| 改進項 | 範例 |
|--------|------|
| 知識補充 | 「分析標準應加入 X 檢查項」 |
| 知識修正 | 「Linear workflow 的狀態流轉有誤」 |
| 新增 reference | 「需要一份新的 X 參考文件」 |

#### 2.4 流程改進

skills 之間的銜接、前後順序、前置檢查需要調整。

| 改進項 | 範例 |
|--------|------|
| 銜接斷裂 | 「A skill 完成後不知道下一步」 |
| 前置缺失 | 「B skill 應先檢查 X 是否存在」 |
| 重複工作 | 「C 和 D skill 做了同樣的事」 |
| 順序不當 | 「應先做 E 再做 F」 |

#### 2.5 新增需求

session 中發現需要但 plugin 目前沒有的能力。

| 改進項 | 範例 |
|--------|------|
| 新 skill | 「需要一個專門做 X 的 skill」 |
| 新 reference | 「需要 Y 的標準參考」 |
| 新範本 | 「需要 Z 文件的範本」 |

### 3. 評估影響與優先級

| 優先級 | 條件 | 行動 |
|--------|------|------|
| P0 | 影響正確性（產出有誤） | 立即修復 |
| P1 | 影響效率（流程卡頓、重複工作） | 下次更新修復 |
| P2 | 影響體驗（格式、順序、提示） | 排入待辦 |
| P3 | 新功能建議 | 評估後決定 |

### 4. 產出回顧報告

```markdown
# Session 回顧報告

**日期**：{YYYY-MM-DD}
**使用的 Skills**：{列表}
**Plugin 版本**：{version}

## 改進摘要

| 類別 | P0 | P1 | P2 | P3 | 合計 |
|------|----|----|----|----|------|
| Skill 內容 | | | | | |
| 範本 | | | | | |
| Reference | | | | | |
| 流程 | | | | | |
| 新增需求 | | | | | |

## 詳細發現

### P0 — 立即修復
{每項列出：問題、證據（對話中的具體片段）、修復方案}

### P1 — 下次更新
{同上}

### P2 — 排入待辦
{同上}

### P3 — 新功能建議
{同上}

## 值得固化的好模式
{這次 session 中效果好的做法，應寫入 skill 或 reference}

## 版本建議
- 當前版本：{version}
- 建議版本：{新版本號}
- 變更摘要：{一句話說明}
```

### 5. 執行修改（--apply 模式）

僅在使用者指定 `--apply` 時執行：

1. **P0 項目**：直接修改對應的 SKILL.md / 範本 / reference
2. **P1 項目**：直接修改，標記 `<!-- retrospective {date}: {改進描述} -->`
3. **P2/P3 項目**：不修改，僅列入報告

修改後：
- 更新 `plugin.json` 的 version（patch +1）
- 執行 `claude plugin validate .` 確認結構正確
- 列出所有變更檔案供使用者確認

### 6. 進化追蹤

在 plugin 根目錄維護 `CHANGELOG.md`，記錄每次回顧驅動的變更：

```markdown
## [{version}] - {YYYY-MM-DD}

### 來源
- Session 回顧

### 變更
- **{skill-name}**: {改進描述}
- **{template-name}**: {改進描述}

### 觸發證據
- {簡述使用者的回饋或發現的問題}
```
