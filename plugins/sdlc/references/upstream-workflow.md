# SDLC 上游階段流程

## 全新功能

```
需求文件 → (驗證) → 系統分析 → 功能規格書 → (驗證) → [交付實作]
  FRD        自動      SAD         BFS+FFS      自動
  PM角色               SA角色      開發者角色
```

## 功能調整

```
調整需求文件 → (驗證) → 系統分析 → 調整規格書 → (驗證) → [交付實作]
  CR-FRD        自動      SAD       CR-BFS+CR-FFS  自動
```

> 驗證通過後交付下游進行實作、測試、審查、提交。

## 各階段職責

| 階段 | 產出 | 角色 | 定義 |
|------|------|------|------|
| 需求文件（FRD / CR-FRD） | 業務需求 | PM | 做什麼、為什麼做 |
| 系統分析（SAD） | 工作流/資訊流/資料流 | SA | 如何實現 |
| 功能規格書（BFS / CR-BFS） | 技術規格 | 開發者 | 怎麼做 |
| 前端規格書（FFS / CR-FFS） | UI 規格 | 開發者 | 畫面怎麼呈現 |
| 規格驗證 | 品質報告 | 架構師 | 能不能做 |

## 步驟間關係

| 步驟 | 前置 | 後續 |
|------|------|------|
| 需求文件 | 無 | 系統分析 |
| 系統分析 | 需求文件 | 功能規格書 |
| 功能規格書 | 系統分析（建議）或需求文件 | 交付實作 |
| 規格驗證 | 任何已完成的文件 | 修復或通過 |
| 舊系統分析 | 舊系統檔案 | 需求文件或系統分析 |
| Linear 開票 | 需求文件（建議） | 開始開發 |
